Processen

Här följer en beskrivning av arbetsprocessen:

Utvecklingsmiljön och Verktygen:

För utvecklingen av webbplatsen har jag använt Visual Studio Code (VS Code) som integrerad utvecklingsmiljö (IDE). För att automatisera och hantera paketanvändningen användes Node.js som runtime-miljö och pakethanteraren Parcel.

Strukturering:

En ny mapp skapades i VS code där alla arbetsfiler organiserades. Mappen innehöll separata underkataloger för HTML, CSS, JavaScript och bilder för att hålla koden organiserad och överskådlig. Detta skapade en tydlig struktur och underlättade hanteringen av olika filtyper.

Paketinstallation med NPM och package.json:

Node Package Manager (NPM) användes för att installera och hantera paket. projektets metadata och dess konfiguration dokumenteras i filen package.json. Detta gör det enkelt för andra utvecklare i större projekt att replikera och installera nödvändiga beroenden.

Utveckling av Webbplatsen:

Utvecklingen av webbplatsen har gjorts med hjälpa av HTML-, CSS- och JavaScript-filer för huvudsidan och två undersidor. Responsiv design implementerades för att säkerställa en bra användarupplevelse på olika enheter och skärmstorlekar.

Automatisering med Parcel:

Parcel användes för att automatisera och underlätta skapandeprocess med hjälpa av local host för omedelbar förhandsgranskning av ändringar. Genom att använda Parcel kan man som utvecklare undvika manuella uppgifter som att konfigurera system och hantera beroenden manuellt. Parcel tar hand om dessa uppgifter automatiskt, vilket sparar tid och minskar risken för fel.

Versionshantering med Git och GitHub:

Projektet versionhanterades med Git, och en GitHub-repositorium skapades för att lagra och hantera koden på distans. Detta möjliggör samarbete och spårning av ändringar över tid. Jag försökte göra regelbundna commits, framför allt vid betydande uppdatering gjordes en commit. Vidare försökte jag skriva tydliga commitmeddelanden för att förklara ändringarna.

Publicering på Netlify:

För att publicera webbplatsen användes Netlify, en molnbaserad plattform för automatiserad deployment och hosting. Netlify integrerades med GitHub-repositoriet, vilket innebar att varje push till master-branchen automatiskt utlöste en ny byggprocess och publicering på Netlify. Detta gjorde det enkelt att distribuera och uppdatera webbplatsen utan manuella steg.

Kontinuerlig Integration och Deployment (CI/CD):

Genom att använda Git, GitHub och Netlify skapades en kontinuerlig integrations- och deploymentprocess. Detta innebar att varje ändring av koden i master-branchen automatiskt byggdes och publicerades på webben, vilket resulterade i en smidig och snabb publiceringsprocess.
